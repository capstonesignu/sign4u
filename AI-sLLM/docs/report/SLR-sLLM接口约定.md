# SLR → sLLM 接口约定

> 目的:规定手语识别(SLR)交给后处理 sLLM 的数据格式。
> 决定:**采用 top-1 单词序列**(候选 + 把握度方式实验后未采纳,见文末)。

## 核心约定:每个词位输出 top-1 单词

SLR 对每个词位只输出**置信度最高的一个单词**,按顺序排列成单词列表交给 sLLM。

## 数据格式(JSON)

```json
{
  "request_id": "001",
  "words": ["배", "아프다", "병원", "가다"]
}
```

- `request_id`:请求标识(便于日志/调试对齐),字符串
- `words`:`list[str]`,每个元素是该位置的 top-1 单词

## sLLM 侧返回

```json
{
  "request_id": "001",
  "normalized_text": "배가 아파서 병원에 갑니다.",
  "status": "success"
}
```

- 输出固定为 **합쇼체**(`~습니다 / ~ㅂ니다`)
- 模型 = `EXAONE-3.5-2.4B-Instruct` 底座 + LoRA 适配器 `exaone-finetuned-v5-cons`(4bit NF4 加载)
- 加载与推理细节见适配器文件夹的 `사용법.md`

## SLR 侧实现(零成本)

SLR 内部本来就在 softmax 后取最大值作为预测,直接把这个值作为 `words[i]` 即可。**不用改模型、不用重训、不用额外接口**。

## 历史:候选 + 把握度方式(已实验,未采纳)

早期设计提议 SLR 输出 `top-3 + 把握度`,由 sLLM 用上下文重选:

```json
"positions": [[["학교", 0.62], ["학원", 0.25], ["회사", 0.11]], ...]
```

实验结果(`exaone-finetuned-v6-cand`):

| 指标 | v5-cons(top-1 训练) | v6-cand(候选格式训练) |
|------|---------------------|----------------------|
| 候选格式 → 正确单词复原率 | — | 79.8%(机制有效) |
| 单一单词输入 → 幻觉率 | 28% | **42%(显著恶化)** |
| 单一单词输入 → 可接受率 | ~69% | ~47% |

**结论**:候选训练让模型把"添加上下文"的倾向带入到了单一单词输入场景,clean-input 质量明显下降。**单一模型无法同时把"候选重选"和"单词整形"两件事都做好**。考虑到 SLR 接口的简化以及 demo 场景以单一单词序列为主,最终决定走 top-1。

候选格式的训练数据生成脚本 `scripts/make_candidate_traindata_v5.py` 与 `exaone-finetuned-v6-cand` 适配器保留作为实验记录,但**不进入交付**。若将来 SLR 能稳定输出置信度且需要重新评估,可作为对照参考。
