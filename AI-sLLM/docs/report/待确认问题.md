# 待确认问题(给组长 / SLR 团队)

> 这些问题需要拍板,才能决定 sLLM 下一步怎么走。按重要性排序。

## A. 接口相关(最关键 —— 决定整个方案走向)

1. **SLR 能否输出 top-k + 把握度?格式如何?**
   - 这决定 sLLM 走哪条路:
     - 只给 1 个最佳词 → sLLM 走"纠错"(实测还原 65%)
     - 给 top-3 + 把握度 → sLLM 走"候选挑词"(实测还原 80%)
2. **候选数确认**:每位置**最多 3 个 + 按把握度阈值动态**(确定的 1 个、模糊的 2–3 个)—— 可以吗?
3. **能否提供真实 SLR 的识别错误/混淆样本?**
   - 现在候选训练数据是**合成的**(随机干扰词)。真实混淆数据(哪些手势容易被认错)会让结果**更可信、更贴近实战**。

## B. 模型 / 任务

4. **模型大小**:7.8B 实测**更差**(更爱幻觉),确认就用 **2.4B**?
5. **语体**:统一 **합쇼체(~습니다)**?还是也要支持 **해요체(~어요)**?(组长原话两个都提过)
6. **时态**:输入无时间词时,统一用**现在式**(가다→갑니다)对吗?

## C. 评测

7. **最终评测标准用什么?**
   - BLEU 对"一个词序对应多种正确说法"的任务**不公平**(分数天生偏低)。
   - 建议:以**人工评测 + 敬语率 + keyword recall(关键词还原率)**为主,BLEU 仅作参考。
8. **数据领域**:测试集偏**医疗**(96 句里 75 句),要不要补**日常/学校/交通**?
