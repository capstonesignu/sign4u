# AI-sLLM 프로젝트 정리

> 한국수어(KSL) 번역 시스템의 **후처리 sLLM** — SLR이 인식한 단어 시퀀스를 자연스러운 한국어(합쇼체) 문장으로 변환한다.

이 문서는 코드 사용법(`README.md`)과 별개로, 프로젝트의 **내용·접근·결과**를 정리한 것이다.

---

## 1. 프로젝트 개요

한국수어를 한국어로 번역하는 전체 시스템에서 **sLLM은 후처리(post-processing) 단계**를 담당한다. 수어 인식(SLR)이 영상에서 뽑아낸 단어(글로스) 시퀀스는 조사·어순·어미가 빠진 거친 형태인데, sLLM이 이를 받아 문법적으로 올바르고 자연스러운 **합쇼체 한국어 문장**으로 다듬는다.

```
수어 영상 → 키포인트 추출 → SLR(수어 인식) → 단어 시퀀스 → [ sLLM ] → 한국어 문장
```

예) `["배", "아프다", "병원", "가다"]` → `"배가 아파서 병원에 갑니다."`

---

## 2. 접근 방법

- **기반 모델**: EXAONE-3.5-2.4B-Instruct (LG, 한국어 특화) + **QLoRA 4bit** 파인튜닝.
- **두 단계 구조 유지**: SLR + sLLM. sLLM을 ‘언어적 문맥으로 인식 오류까지 보정하는 계층’으로 설계 (음성 인식의 언어 모델 보정과 유사한 역할).
- **7단계 제어 추론 파이프라인**: 생성 → 제약 채점 → 후보 선택 → 형식 정리 → 의미 정리(환각 제거 포함) → 가드레일 → 트레이스. (세부는 `README.md` 참고)
- **학습 데이터**: 의료·일상 도메인의 (단어 시퀀스 → 합쇼체 문장) 쌍 약 8천 문장.

## 3. 조장 제시 3개 과제 결과

### 과제 #2 — 말투 공손화 (합쇼체) ✅ 완료
모델 출력을 모두 합쇼체(`~습니다 / ~ㅂ니다`)로 통일했다. 합쇼체율 **v4 97.9% → 최신 v5 100%**.

### 과제 #3 — 더 큰 모델 ✅ 완료 (결론: 2.4B 유지)
EXAONE-3.5 **7.8B**를 같은 데이터로 파인튜닝해 비교했으나, **오히려 환각이 더 많고 충실도가 낮았다** (BLEU 0.18 vs 2.4B 0.28). 큰 모델이 ‘자유롭게 살을 붙이는’ 경향이 강해, ‘충실 + 최소 변환 + 고정 어체’라는 제약 과제에는 불리했다. → **데이터로 뒷받침된 결정으로 2.4B 4bit를 채택.** (같은 모델에서 4bit가 16bit보다도 품질이 좋았음 — QLoRA 어댑터가 4bit 베이스에 맞춰 학습되기 때문.)

### 과제 #1 — 단어 후보 (top-k) ✅ 실험 완료 (결론: 채택하지 않음)
SLR이 위치마다 단일 단어 대신 **후보 top-3 + 확신도**를 주면 sLLM이 문맥으로 옳은 단어를 고를 수 있는지 검증했다. 후보 형식으로 파인튜닝(`exaone-finetuned-v6-cand`) 시 정답 복원율 **약 79.8%** — 단일 추측(65.3%)보다 크게 향상되어 메커니즘 자체는 유효함을 확인했다.

**그러나** 후보 학습은 **단일 단어 입력에서의 품질을 떨어뜨린다**: clean 입력 기준 환각률이 **v5-cons 28% → v6-cand 42%**로 악화. "후보 처리"와 "단일 단어 정제"를 한 모델로 동시에 잘하기 어려운 **명확한 trade-off**가 데이터로 드러났다.

이 결과와 SLR 측 인터페이스의 단순성을 고려해 **최종 배포는 top-1 단어 시퀀스 입력 + `v5-cons` 단일 모델**로 확정했다. 후보 방식은 채택하지 않으나, 메커니즘 검증 결과는 향후 SLR이 신뢰도까지 제공할 수 있게 될 때를 위한 자료로 남긴다.

## 4. 환각(hallucination) 개선 — v5

모델이 ‘입력에 없는 단어를 지어내는’ 환각이 주요 약점이었다. 원인을 분석한 결과, **학습 데이터 자체에 ‘입력에 없는 내용을 덧붙이던’ 문장 587개**가 섞여 있어 모델이 그것을 그대로 학습한 것이 확인됐다(시스템 프롬프트는 ‘추가 금지’를 명시하는데도 목표 문장이 이를 어기고 있었다).

이를 정제한 뒤 재학습한 **`v5-cons`** 가 현재 최선 모델이다.
- 학습 데이터: 8,072 → 정제 후 **7,485문장**
- 결과: 환각 감소 · 영어 혼입 **0%** · 합쇼체율 **100%**

## 5. 평가

- **재현 가능한 평가 도구** `scripts/reaudit.py`를 구축했다 — GPU 없이 kiwi 형태소 기준으로 환각·띄어쓰기를 **코드로** 측정해, 보고서 수치를 누구나 재현·검증할 수 있게 했다.
- **LLM 심사 수용 가능률**(충실성 + 자연스러움 + 합쇼체) 약 **69%**. 단순 exact-match(7%)가 가리던 실제 품질을 드러낸 것으로, ‘오답’의 상당수는 정답과 표현만 다른 **유효 패러프레이즈**였다.
- 단일 정의의 환각 지표로 다시 감사하면, 높게 잡히던 수치의 약 70%가 **동의어·형태 변화·합성어 등 지표 오탐**이며, **실제 환각은 약 7~8% 수준**이다.

## 6. SLR 인터페이스

sLLM은 **각 위치별 top-1 단어 시퀀스**를 입력으로 받는다.

```json
{"request_id": "001", "words": ["배", "아프다", "병원", "가다"]}
```

- 후보 top-3 + 확신도 인터페이스도 실험했으나(§3 과제 #1) 단일 단어 입력 품질을 떨어뜨리는 trade-off가 확인되어 **채택하지 않았다**.
- top-1 단어 시퀀스만으로도 sLLM이 합쇼체 정제와 문맥 보정을 수행하며, 현 LLM-심사 수용 가능률 **약 69%** 를 달성한다.
- SLR ↔ sLLM 간 인터페이스 상세는 `docs/report/SLR-sLLM接口约定.md` 참고.

## 8. 남은 과제 / 향후

- **반의어/극성 오류** 소수 케이스(예: 춥다 → 안 춥다, 싱겁다 → 짜다) — 의료 맥락에서 위험하므로 다음 개선 대상.
- **팀 데모에서 보고된 환각 사례 검증** — `v5-cons`로 동일 입력을 재현해 잔여 환각률을 측정하고 개선 우선순위를 정한다.
- 도메인 확장(의료 외 일상/학교/교통).
